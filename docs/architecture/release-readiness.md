# Release Readiness

Paper Forge Phase 1 is targeting a public macOS MVP with three launch features:

1. PDF to Images
2. TXT to PDF
3. Flatten PDF

## Current Build Status

The current local preview build is:

- native SwiftUI app
- ad-hoc signed locally unless `PAPER_FORGE_SIGN_IDENTITY` is provided
- not notarized yet
- packageable as a local `.dmg`
- generated by `packaging/macos/build_native_app.sh`
- packaged by `packaging/macos/build_release_dmg.sh`
- validated by `packaging/macos/validate_release.sh`

Ad-hoc signed outputs are suitable for development preview only.

Current machine status:

- no valid code-signing identities were found by `security find-identity -v -p codesigning`
- public distribution requires installing an Apple Developer ID Application certificate

## Direct Public Launch Requirements

Before public direct-download release:

- create release bundle metadata for the final app name and bundle identifier
- add a production app icon
- replace the deterministic MVP icon with a polished brand icon before broad launch
- sign the `.app` with a Developer ID Application certificate
- enable hardened runtime
- notarize the signed app with Apple
- package the notarized app into a `.dmg`
- sign the `.dmg`
- document minimum supported macOS version
- verify the app opens on a clean macOS user account
- verify all Phase 1 workflows with files outside the project directory

## App Store Considerations

Before submitting to the Mac App Store:

- create a real Xcode project or equivalent App Store-compatible build pipeline
- configure a Mac App Store bundle identifier and provisioning profile
- add sandbox entitlements
- use security-scoped bookmarks or document picker access for user-selected files
- avoid writing outside user-approved locations
- review whether `NSWorkspace.open` behavior is acceptable under sandboxing
- provide app category, screenshots, privacy details, support URL, and marketing copy
- confirm no private APIs or unsupported helper behavior are used
- run the app in a sandboxed build before submission

## Image Generation Status

The current Codex session does not expose a callable image generation tool, so the MVP icon is generated locally and deterministically from native drawing code.

Use a designer-produced icon or a future image-generation pass before final public marketing launch.

## MVP Launch Gate

Phase 1 MVP should not expand feature scope until these gates pass:

- app remains responsive during conversion
- cancellation can be requested during long jobs
- progress updates are visible
- all three launch modules pass smoke tests
- local app bundle builds reproducibly
- packaging plan for signing/notarization is documented
